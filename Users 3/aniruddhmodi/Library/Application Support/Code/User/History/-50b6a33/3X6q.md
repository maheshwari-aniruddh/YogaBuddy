# Publishing Roze to the Apple App Store

## Prerequisites

### 1. Apple Developer Account
- **Cost:** $99/year
- **Sign up:** https://developer.apple.com/programs/
- **What you get:** Ability to publish apps, TestFlight access, app analytics

### 2. Required Assets

#### Screenshots (Required)
You need screenshots for different iPhone sizes:
- **iPhone 6.7"** (iPhone 15 Pro Max, 14 Pro Max, etc.): 1290 x 2796 pixels
- **iPhone 6.5"** (iPhone 11 Pro Max, XS Max): 1242 x 2688 pixels
- **iPhone 5.5"** (iPhone 8 Plus): 1242 x 2208 pixels

**Tip:** Take 3-5 screenshots showing:
1. Login/Welcome screen
2. Today/Journal entry screen
3. Calendar view with entries
4. Statistics dashboard
5. Settings or special features

#### App Preview Video (Optional but recommended)
- 15-30 seconds showing app features
- Same sizes as screenshots

#### App Icon (Already done! ✅)
- Your `app_icon.png` is already configured

## Step-by-Step Publishing Process

### Step 1: Prepare Your Xcode Project

1. **Open the iOS project in Xcode:**
```bash
open ios/Runner.xcworkspace
```

2. **In Xcode, select the Runner target and update:**
   - **Bundle Identifier:** Change to something unique like `com.yourname.roze`
   - **Version:** 1.0.0
   - **Build:** 1
   - **Display Name:** Roze
   - **Team:** Select your Apple Developer team

3. **Update Signing:**
   - Go to "Signing & Capabilities"
   - Check "Automatically manage signing"
   - Select your team

### Step 2: Build the Release Version

Run this command to build the iOS release:

```bash
flutter build ipa
```

This creates an `.ipa` file at: `build/ios/ipa/`

### Step 3: Create App in App Store Connect

1. Go to: https://appstoreconnect.apple.com/
2. Click "My Apps" → "+" → "New App"
3. Fill in:
   - **Platform:** iOS
   - **Name:** Roze
   - **Primary Language:** English
   - **Bundle ID:** (select the one you created)
   - **SKU:** Can be anything unique (e.g., `roze-001`)
   - **User Access:** Full Access

### Step 4: Upload App to App Store Connect

**Option A: Using Xcode (Easiest)**

1. In Xcode, select "Any iOS Device (arm64)" as the target
2. Go to: Product → Archive
3. Wait for archive to complete
4. In the Organizer window, click "Distribute App"
5. Select "App Store Connect" → Next
6. Select "Upload" → Next
7. Select your signing options → Next
8. Click "Upload"

**Option B: Using Transporter App**

1. Download "Transporter" from Mac App Store
2. Open Transporter
3. Sign in with your Apple ID
4. Drag and drop your `.ipa` file
5. Click "Deliver"

### Step 5: Complete App Store Listing

In App Store Connect:

1. **App Information:**
   - Add your subtitle
   - Select categories
   - Add content rights info

2. **Pricing and Availability:**
   - Select countries (or worldwide)
   - Price: Free or set a price
   - Availability date

3. **App Privacy:**
   - Fill out privacy questionnaire
   - Add your privacy policy URL
   - Declare what data you collect:
     - ✅ User account/profile (Google Sign-In)
     - ✅ User content (journal entries, videos)
     - ✅ Usage data (Firebase Analytics)

4. **Version Information:**
   - Add screenshots (drag and drop)
   - Write your description
   - Add keywords
   - Support URL
   - Marketing URL (optional)
   - Version: 1.0.0
   - Copyright: © 2025 Your Name
   - Release notes: "Initial release of Roze"

5. **Build:**
   - Click "+" next to Build
   - Select the build you uploaded
   - Answer export compliance questions

6. **Age Rating:**
   - Complete the questionnaire
   - Should be 4+ for Roze

### Step 6: Submit for Review

1. Review all information
2. Click "Add for Review"
3. Click "Submit to App Review"

**Review Time:** Usually 1-3 days

## Testing Before Submission (Recommended)

### Use TestFlight for Beta Testing

1. In App Store Connect, go to TestFlight tab
2. Add internal testers (up to 100 people)
3. Upload a build
4. Invite testers via email
5. They download TestFlight app and test your app

**Why TestFlight?**
- Test on real devices without publishing
- Get feedback before public release
- No review needed for internal testing

## Common Rejection Reasons & How to Avoid

1. **Missing Privacy Policy:** Make sure you add one!
2. **Incomplete Information:** Fill out ALL fields
3. **Poor Screenshots:** Use actual app screenshots, not mockups
4. **Crash on Launch:** Test thoroughly before submitting
5. **Login Issues:** Make sure Google Sign-In works perfectly
